API Reference

Introduction

The CRM System API allows you to programmatically access and manipulate your CRM data.

Authentication

All API requests require authentication using an API key. You can generate an API key in your account settings.

curl -H "Authorization: Bearer YOUR_API_KEY" https://api.crmsystem.com/v1/customers

Endpoints

Customers

  • GET /api/v1/customers - List all customers
  • GET /api/v1/customers/{id} - Get a specific customer
  • POST /api/v1/customers - Create a new customer
  • PUT /api/v1/customers/{id} - Update a customer
  • DELETE /api/v1/customers/{id} - Delete a customer

Deals

  • GET /api/v1/deals - List all deals
  • GET /api/v1/deals/{id} - Get a specific deal
  • POST /api/v1/deals - Create a new deal
  • PUT /api/v1/deals/{id} - Update a deal
  • DELETE /api/v1/deals/{id} - Delete a deal

Users

  • GET /api/v1/users - List all users
  • GET /api/v1/users/{id} - Get a specific user
  • POST /api/v1/users - Create a new user
  • PUT /api/v1/users/{id} - Update a user
  • DELETE /api/v1/users/{id} - Delete a user

Rate Limiting

The API is rate limited to 100 requests per minute per API key. If you exceed this limit, you will receive a 429 Too Many Requests response.

Error Handling

The API returns standard HTTP status codes to indicate success or failure of a request. Error responses include a JSON body with an error message.

{
  "error": "Invalid customer ID",
  "status": 404
}
